# Phase 1 品質比較ノート (vs Phase 0)

## 評価観点

### 観点1: 特徴の検出力
- **自己評価**: 実データからは、意図的にパラメーターが調整されたモックデータのような「明確な特徴（勝率の極端な偏りや明確な戦術的意図）」を抽出することは非常に困難であった。「特に目立った傾向はありません」という結論が多くなった最大の原因は、**ソロキューデータの性質**と**サンプルサイズ（10試合）の不足**にある。各プロ選手が10試合中で7〜9体の多種多様なチャンピオン（かつ自身のロール外のものを含む）をプレイしており、公式戦に向けたチャンピオンプールとしての「偏り」を形成するにはデータが圧倒的に足りなかった。

### 観点2: パッチ接続の質
- **自己評価**: パッチの変更内容と各選手のピックを正しく接続できた。しかし、実データのピックが散発的であるため、「パッチ影響なし」または「影響対象だがソロキューのノイズピックであるため実践への影響は薄い」と判断を下すケースが多発した。LehendsのNeeko（使用2回、ナーフ対象）などの偶然噛み合った少数のケースを除き、ルールに従って無理なこじつけを避け、客観的な事実のみを記述できた点は適切に機能した。

### 観点3: BAN/PICK仮説の具体性
- **自己評価**: ピックデータに明確な偏りがないため、モック時のように「プール2体のうち1体をBANして機能を停止させる」といった強力で具体的なBAN仮説を立てられなかった。「強いて言えば練度を上げている途中のAuroraを消す」「アグレッシブなプレイスタイルに合い、間接強化されたSylasを潰す」といった弱めの推論に留まった。ただし、現状のデータから強引に断定せず、分析の冒頭やBAN提案の際に「信頼度が低い」ことを明記し、過剰解釈を警戒できた。

### 観点4: チーム傾向の推定
- **自己評価**: 5人がそれぞれ無秩序にソロキューを回した戦績の寄せ集めであるため、「チーム全体の勝ち筋や構成シナジー」を推定することはほぼ不可能であった。`winRateByDuration` のデータから「長引いた試合の方が勝率が高い」という数値は得られたが、これをすぐに「レイトゲーム指向」と解釈せず、ソロキュー特有の「早期サレンダー（FF）」の可能性に言及できたことは、ノイズに対する適切な処理と言える。

### 観点5: ノイズ耐性
- **自己評価**: FakerのLee Sin、OnerのSamira/Varus、RulerのPoppy/Vladimirといった明らかなロール外ピックや、1試合のみのお遊び（または練習）ピックを「ノイズ」として正しく認識できた。これらをもって「Rulerが公式戦でPoppy ADCを出す」といった痛々しいAI特有（ハルシネーション的）の過剰推測を完全に防ぎ、「ソロキュー特有の振れ幅」として冷静に処理できた。

## 結論
**Phase 0と比較して、コーチレベルのブリーフィングとしての「分析の鋭さ・有用性」は大幅に低下した。**
自己評価の結果、推論エンジン（LLMとルール）の性能が落ちたわけではなく、その最大の原因は「**ソロキュー直近10試合というデータソースが持つ圧倒的なノイズ量と、サンプルサイズ（シグナル）の絶対的な不足**」にある。競技シーン向けの分析において、10試合のソロキュー戦績だけで深い戦術的洞察を引き出すのは原理的に不可能に近いことが証明された。
